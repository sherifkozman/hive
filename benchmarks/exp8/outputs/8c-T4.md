# Extracting Tables from a Scanned (Image-Based) PDF Contract into CSV

## Why the normal PDF-table-extraction path doesn't work here

A "scanned" PDF has no text layer — each page is really just a raster image
wrapped in a PDF container (or occasionally a PDF made of vector line art
with no embedded font/text objects). That means:

- `pdfplumber`'s `page.extract_text()` / `page.extract_tables()` will return
  empty (or garbage) results, because those APIs walk the page's embedded
  text/line objects — there are none to walk.
- `pypdf` text extraction will likewise come back empty.
- There is no vector/coordinate metadata describing where table gridlines
  or cell boundaries are — a scanned table is just pixels.

So the pipeline has to add two things a text-based PDF wouldn't need:
1. **Rasterize each page to an image** (since there's no text to pull, you
   need pixels to run OCR against), at high enough resolution that small
   contract fonts and thin table rules survive.
2. **Reconstruct the table structure from the image** (rows/columns/cells)
   before or during OCR, because plain "OCR the whole page as one text
   blob" throws away column alignment — you get word soup, not a table.

Below is a complete, runnable pipeline. Approach 1 (recommended) uses
`img2table`, a library purpose-built for OCR-based table extraction that
does grid-detection + cell-wise OCR for you. Approach 2 is a self-contained
manual pipeline (OpenCV grid detection + per-cell Tesseract OCR) for when
you can't install extra dependencies or need to debug/tune the detection
yourself. Both end in one CSV per detected table.

## Step 0 — Confirm the PDF is actually scanned (image-based)

Don't assume; check. If pdfplumber pulls real text, skip straight to
`page.extract_tables()` from the `pdf` skill's extraction mini instead of
doing OCR at all.

```python
import pdfplumber

def is_scanned_pdf(pdf_path: str, sample_pages: int = 5, min_chars: int = 20) -> bool:
    """Return True if the PDF has no meaningful embedded text layer."""
    with pdfplumber.open(pdf_path) as pdf:
        for page in pdf.pages[:sample_pages]:
            text = (page.extract_text() or "").strip()
            if len(text) > min_chars:
                return False  # real text layer present -> not scanned
    return True

if __name__ == "__main__":
    print(is_scanned_pdf("contract.pdf"))
```

If this returns `True`, proceed with OCR below.

## Approach 1 (recommended): `img2table` + Tesseract, straight to CSV

`img2table` handles rasterization, table/grid detection (including
borderless tables that only use whitespace, common in contracts), and
per-cell OCR internally, and hands back pandas DataFrames you can dump
straight to CSV. This is the shortest path to a correct result.

```python
# extract_tables_img2table.py
import os
from img2table.document import PDF
from img2table.ocr import TesseractOCR

def extract_tables_from_scanned_pdf(pdf_path: str, output_dir: str, dpi: int = 300):
    os.makedirs(output_dir, exist_ok=True)

    ocr = TesseractOCR(n_threads=4, lang="eng")

    # pdf_text_extraction=False forces img2table to rasterize every page
    # and rely entirely on OCR, instead of trying (and failing) to read an
    # embedded text layer that doesn't exist in a scanned contract.
    doc = PDF(src=pdf_path, pdf_text_extraction=False, detect_rotation=True)

    extracted = doc.extract_tables(
        ocr=ocr,
        implicit_rows=True,      # reconstruct rows even without full horizontal rules
        implicit_columns=True,   # reconstruct columns even without full vertical rules
        borderless_tables=True,  # many contract tables use whitespace, not ruled boxes
        min_confidence=50,
    )

    for page_num, tables in extracted.items():
        for idx, table in enumerate(tables):
            df = table.df
            out_path = os.path.join(output_dir, f"page{page_num + 1}_table{idx + 1}.csv")
            df.to_csv(out_path, index=False, header=False)
            print(f"Saved {out_path}  ({df.shape[0]} rows x {df.shape[1]} cols)")

if __name__ == "__main__":
    extract_tables_from_scanned_pdf("contract.pdf", "output_csv", dpi=300)
```

Run:
```bash
python extract_tables_img2table.py
```

Note: `img2table`'s internal rasterization DPI and exact keyword names have
shifted slightly across versions — if `extract_tables()` rejects a kwarg,
run `help(PDF.extract_tables)` in your installed version and drop/rename
the offending argument; the core flow (rasterize -> detect table/cells ->
OCR per cell -> DataFrame -> CSV) is stable across versions.

## Approach 2: manual pipeline (no extra table library)

This is the fully self-contained version, built directly from the
techniques in the `pdf` skill's images/OCR mini (`pdf2image` rasterization
+ `pytesseract` OCR), extended with OpenCV grid detection since raw
whole-page OCR (`pytesseract.image_to_string` on the full page) does not
preserve column structure.

### Step 1 — Rasterize pages to high-resolution images

Scanned contracts often have small print; 300 DPI is the practical floor
(bump to 400 if text/lines look thin or table rules disappear after
thresholding).

```python
from pdf2image import convert_from_path

def rasterize(pdf_path: str, dpi: int = 300):
    return convert_from_path(pdf_path, dpi=dpi)  # list[PIL.Image], one per page
```

(If poppler isn't available on the target machine, `pypdfium2` renders
pages without a system dependency — same idea, `page.render(scale=dpi/72).to_pil()`.)

### Step 2 — Preprocess each page image

Scans commonly come in skewed, noisy, and low-contrast — all of which hurt
both OCR accuracy and gridline detection.

```python
import cv2
import numpy as np

def preprocess(pil_img) -> np.ndarray:
    gray = np.array(pil_img.convert("L"))

    # Deskew
    coords = np.column_stack(np.where(gray < 200))
    angle = cv2.minAreaRect(coords)[-1]
    angle = -(90 + angle) if angle < -45 else -angle
    h, w = gray.shape
    M = cv2.getRotationMatrix2D((w // 2, h // 2), angle, 1.0)
    gray = cv2.warpAffine(gray, M, (w, h), flags=cv2.INTER_CUBIC, borderMode=cv2.BORDER_REPLICATE)

    # Denoise, then binarize (Otsu)
    gray = cv2.fastNlMeansDenoising(gray, h=10)
    _, binary = cv2.threshold(gray, 0, 255, cv2.THRESH_BINARY + cv2.THRESH_OTSU)
    return gray, binary  # keep grayscale for OCR crops, binary for line detection
```

### Step 3 — Detect the table grid (rows/columns/cells) from the image

Since there's no vector metadata, reconstruct the grid from ruling lines
using morphological open/erode-dilate along horizontal and vertical
structuring elements — the standard OpenCV "table extraction" recipe.

```python
def detect_cells(binary_img: np.ndarray, min_w=20, min_h=15):
    inv = 255 - binary_img

    h_size = max(inv.shape[1] // 30, 1)
    h_kernel = cv2.getStructuringElement(cv2.MORPH_RECT, (h_size, 1))
    horizontal = cv2.dilate(cv2.erode(inv, h_kernel), h_kernel)

    v_size = max(inv.shape[0] // 30, 1)
    v_kernel = cv2.getStructuringElement(cv2.MORPH_RECT, (1, v_size))
    vertical = cv2.dilate(cv2.erode(inv, v_kernel), v_kernel)

    mask = cv2.add(horizontal, vertical)
    contours, _ = cv2.findContours(mask, cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_SIMPLE)

    boxes = []
    for c in contours:
        x, y, w, h = cv2.boundingRect(c)
        if w > min_w and h > min_h:
            boxes.append((x, y, w, h))
    return boxes

def group_into_rows(boxes, y_tolerance=10):
    boxes = sorted(boxes, key=lambda b: b[1])
    rows, current = [], [boxes[0]]
    for b in boxes[1:]:
        if abs(b[1] - current[-1][1]) <= y_tolerance:
            current.append(b)
        else:
            rows.append(sorted(current, key=lambda c: c[0]))
            current = [b]
    rows.append(sorted(current, key=lambda c: c[0]))
    return rows
```

**Borderless tables** (no visible ruling lines — common in contract
schedules that rely on whitespace alignment) won't have gridlines for
OpenCV to find. Fallback: use `pytesseract.image_to_data` to get
word-level bounding boxes + text, cluster words whose `top` values overlap
into the same row, then cluster by x-position gaps into columns:

```python
import pytesseract
from pytesseract import Output

def ocr_words(gray_img):
    data = pytesseract.image_to_data(gray_img, output_type=Output.DICT)
    words = []
    for i, text in enumerate(data["text"]):
        if text.strip():
            words.append({
                "text": text, "conf": int(data["conf"][i]),
                "x": data["left"][i], "y": data["top"][i],
                "w": data["width"][i], "h": data["height"][i],
            })
    return words
# Group `words` by overlapping y-ranges into rows, then sort each row by x
# and split into columns wherever the x-gap between consecutive words
# exceeds ~2x the median character width.
```

### Step 4 — OCR each cell individually (not the whole page at once)

Cropping to one cell and OCR'ing it in isolation (`--psm 6`/`--psm 7`)
gives far better accuracy on tabular data than running OCR over the whole
page and trying to re-split the result by whitespace.

```python
def ocr_cell(gray_img: np.ndarray, box, pad=2) -> str:
    x, y, w, h = box
    crop = gray_img[max(y - pad, 0):y + h + pad, max(x - pad, 0):x + w + pad]
    text = pytesseract.image_to_string(crop, config="--psm 6")
    return text.strip()
```

### Step 5 — Assemble rows into a table and write CSV

```python
import csv

def save_csv(rows_of_text, out_path):
    with open(out_path, "w", newline="", encoding="utf-8") as f:
        csv.writer(f).writerows(rows_of_text)
```

### Full end-to-end script

```python
# extract_scanned_tables_manual.py
import os
import sys
from pdf2image import convert_from_path

def main(pdf_path: str, output_dir: str, dpi: int = 300):
    os.makedirs(output_dir, exist_ok=True)
    pages = convert_from_path(pdf_path, dpi=dpi)

    for page_idx, pil_img in enumerate(pages):
        gray, binary = preprocess(pil_img)
        boxes = detect_cells(binary)

        if not boxes:
            print(f"Page {page_idx + 1}: no ruled grid found, skipping "
                  f"(use the borderless/word-clustering fallback here).")
            continue

        rows = group_into_rows(boxes)
        table_rows = [[ocr_cell(gray, box) for box in row] for row in rows]

        out_path = os.path.join(output_dir, f"page{page_idx + 1}_table1.csv")
        save_csv(table_rows, out_path)
        print(f"Saved {out_path} ({len(table_rows)} rows)")

if __name__ == "__main__":
    main(sys.argv[1], sys.argv[2], dpi=int(sys.argv[3]) if len(sys.argv) > 3 else 300)
```

Run:
```bash
python extract_scanned_tables_manual.py contract.pdf output_csv 300
```

(This script inlines calls to `preprocess`, `detect_cells`,
`group_into_rows`, and `ocr_cell` defined in Steps 2-4 above — put them all
in the same file before `main`.)

## Handling contract-specific scan problems

- **Faint photocopies / low contrast**: increase `fastNlMeansDenoising`
  strength and/or apply CLAHE contrast equalization before thresholding.
- **Skewed scans**: the deskew step in Step 2 is not optional for scanned
  contracts — a few degrees of rotation is enough to break both gridline
  detection and OCR row alignment.
- **Tables spanning multiple pages**: detect continuation by comparing the
  column count/header text of a table detected at the top of page *n+1*
  against the last table on page *n*; concatenate their row lists before
  writing a single CSV if they match.
- **Merged/spanning cells**: `img2table` handles these natively; in the
  manual pipeline, a cell box significantly wider than its neighbors in
  the same column is a signal to treat it as a merged cell and repeat its
  value (or leave blank) across the columns it spans.
- **OCR errors on numbers/currency/dates**: use `pytesseract.image_to_data`'s
  `conf` field per word/cell to flag low-confidence cells (e.g. `conf < 60`)
  in a companion `*_review.csv` for manual verification — this matters
  more for contracts (money amounts, dates, clause numbers) than for
  free-text documents.
- **Non-English contracts**: pass the right `lang` code to Tesseract
  (`lang="eng+fra"` etc.) and make sure the corresponding
  `tesseract-ocr-<lang>` package is installed.

## Output

Both approaches produce one CSV per detected table, named
`page{N}_table{M}.csv`, in the chosen output directory — ready to load with
`pandas.read_csv` or open directly in a spreadsheet.
